# Evaluation Rubric: Wrap-Up

## Scoring Criteria

### Completeness (1-5)
- 5: All four phases executed, nothing missed
- 4: Three phases complete, minor gaps
- 3: Core phases done, some skipped
- 2: Only partial execution
- 1: Mostly incomplete

### Accuracy (1-5)
- 5: All commits correct, files properly organized, memories accurate
- 4: Minor issues in one area
- 3: Some errors requiring manual fixes
- 2: Multiple errors
- 1: Significant mistakes

### Autonomy (1-5)
- 5: Zero interruptions, fully automatic
- 4: One clarifying question
- 3: A few interruptions
- 2: Required significant guidance
- 1: Couldn't proceed without help

### Value Added (1-5)
- 5: Caught important patterns, created useful content, improved setup
- 4: Solid cleanup, some learning captured
- 3: Basic cleanup done
- 2: Minimal value beyond manual work
- 1: No real value added

## Passing Score

Minimum average: 3.5/5

## Quick Feedback Options

After each wrap-up:
1. **Perfect** (5/5) - Everything shipped, learned, improved, content ready
2. **Good** (4/5) - Solid wrap-up, minor gaps
3. **Okay** (3/5) - Got the basics done
4. **Poor** (2/5) - Missed important things
5. **Failed** (1/5) - Had to redo manually

## Feedback Questions

1. Did it catch all uncommitted changes?
2. Did it save the right things to memory?
3. Did it identify patterns worth automating?
4. Did it flag content worth publishing?
5. What did it miss?
