# Evaluation Rubric: Code Review

## Scoring Criteria

### Issue Detection (1-5)
- 5: Caught all bugs, security issues, and performance problems
- 4: Caught critical/high issues, missed some medium/low
- 3: Caught most critical issues, missed some high priority
- 2: Missed critical issues
- 1: Failed to identify obvious problems

### Actionability (1-5)
- 5: All recommendations are specific with code examples
- 4: Most recommendations actionable, some vague
- 3: Mix of actionable and vague recommendations
- 2: Mostly vague recommendations
- 1: Recommendations not actionable

### Prioritization (1-5)
- 5: Issues perfectly prioritized by severity and impact
- 4: Good prioritization, minor misorderings
- 3: Acceptable prioritization
- 2: Poor prioritization (low issues marked high or vice versa)
- 1: No meaningful prioritization

### Tone (1-5)
- 5: Constructive, professional, encouraging
- 4: Professional with minor tone issues
- 3: Neutral but not encouraging
- 2: Overly critical or harsh
- 1: Unprofessional or demotivating

### Completeness (1-5)
- 5: Covered all review areas (correctness, security, performance, maintainability)
- 4: Covered most areas thoroughly
- 3: Some areas well-covered, others shallow
- 2: Major gaps in coverage
- 1: Incomplete review

## Passing Score

Minimum average: 3.5/5

## Quick Feedback Options

After each code review:
1. **Perfect** (5/5) - Caught everything, great recommendations
2. **Good** (4/5) - Caught important issues, minor gaps
3. **Okay** (3/5) - Useful but missed some things
4. **Poor** (2/5) - Missed important issues
5. **Failed** (1/5) - Unhelpful review

## Feedback Questions

For detailed feedback:
1. Did the review catch the issues you already knew about?
2. Did the review find issues you didn't know about?
3. Were the recommendations actionable?
4. What did the review miss?
5. What would make this review more useful?
