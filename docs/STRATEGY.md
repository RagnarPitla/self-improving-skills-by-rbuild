# Strategy: Become the Top Skills Repo

**Goal:** Make `self-improving-skills` the most forked, most starred skills repository on GitHub.

---

## Why This Can Win

### The Gap in the Market

| Existing Repos | What They Do | What's Missing |
|----------------|--------------|----------------|
| awesome-prompts | Curated prompt lists | No evolution, no structure |
| langchain | LLM orchestration | Too complex for skills |
| OpenBrain | Practitioner skills | No self-improvement mechanism |
| cursor-rules | IDE-specific rules | Single platform, static |

**Our differentiation:** Skills that get better over time. Nobody else is doing this.

### Why People Will Fork

1. **Immediate value** - Copy skills, use today
2. **Compounding value** - Skills improve with use
3. **Export anywhere** - Not locked to one platform
4. **Community effect** - Learnings shared back

## Risk Mitigation

| Risk | Mitigation |
