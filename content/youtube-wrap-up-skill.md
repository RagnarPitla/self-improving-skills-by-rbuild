## Script

### HOOK [0:00-0:30]

*[Direct to camera]*

I've built a lot of Claude Code skills. Some clever. Some over-engineered. Some I forgot existed.

The one I actually use every single session? It's basically a glorified checklist.

*[Beat]*

And it's the most valuable thing I've built. Because it makes Claude get smarter every time I use it.

Let me show you.

---

### SECTION 1: THE PROBLEM [0:30-1:30]

*[Screen share or B-roll of messy session]*

Here's what used to happen at the end of every coding session:

I'd do an hour of work. Close the laptop. Feel productive.

Next day I'd open the project and realize... I never committed anything. Nothing got pushed. If my laptop died, that work was gone.

*[Cut back to camera]*

And that's just the obvious problem.

The less obvious one? All the stuff I learned during that session - debugging insights, project quirks, patterns I discovered - all of that just... vanished. Lost in a conversation I'd never reopen.

I'd make the same mistakes next time. Ask Claude the same questions. Start from scratch every session.

*[Frustrated gesture]*

That's not how compounding works.

---

### SECTION 2: THE SKILL [1:30-4:00]

*[Screen share: showing SKILL.md]*

So I built wrap-up. A four-phase skill that runs at the end of every session.

**Phase 1: Ship It**

*[Show git status, commit, push]*

This is the "I forgot to commit" fix. Claude runs git status on every repo I touched. If there's uncommitted changes, it commits with a descriptive message and pushes.

It also organizes files. If I created a markdown file in the wrong directory, it moves it. If I forgot to name something properly, it renames it.

Boring? Yes. Valuable? Also yes.

*[Cut to camera]*

**Phase 2: Remember It**

This is where the compounding starts.

*[Show memory hierarchy diagram]*

Claude has this whole memory hierarchy - CLAUDE.md, rules, auto memory, local notes. Most people never use it deliberately.

This phase forces a review. "Did we learn anything that should persist? Where should it live?"

Over weeks, my setup gets smarter without me thinking about it. Patterns I discovered months ago are still helping today because they got saved to the right place.

**Phase 3: Review & Apply**

*[Cut to camera, lean in]*

This one surprised me. I added it half-expecting it to be useless.

But Claude actually catches real patterns.

*[Mock Claude voice]*
"You asked me to use TypeScript types three times today. That should be automatic."

Then it writes the rule. Commits it. Next session, that mistake is gone.

*[Normal voice]*

Self-improving tooling with zero effort from me.

**Phase 4: Publish It**

The newest phase.

Turns out a lot of sessions produce something worth sharing - an interesting problem solved, a pattern discovered, a conversation worth turning into content.

I just never got around to publishing it. Lost in conversations I'd never reopen.

Now Claude flags it, drafts it, saves it to a folder. I still decide whether to post, but the draft exists instead of the idea dying in my head.

---

### SECTION 3: THE META POINT [4:00-5:30]

*[Direct to camera, slower pace]*

Here's what I've realized.

The best skills aren't impressive. They don't generate fancy demos. They don't make people say "wow" on Twitter.

The best skills run the boring routines you'd skip.

*[Count on fingers]*

Commit code? I'd skip it.
Save learnings? I'd skip it.
Review mistakes? I'd skip it.
Draft content? I'd skip it.

Now I just type `/wrap-up` and all of it happens.

*[Pause]*

And here's the thing about compounding. It's invisible at first.

Session one, wrap-up saves one pattern. Session ten, there's ten patterns. Session fifty, Claude handles edge cases I forgot existed because they got captured and automated along the way.

That's not a demo. That's infrastructure. That's how you actually get value from AI over time.
