## Thread

### Tweet 2

Introducing Self-Improving Skills by @RBuild_ai

A framework where skills learn, evolve, and compound over time.

Run a skill → Rate the output → Skill creates a new version

Every use makes it better.

github.com/RagnarPitla/self-improving-skills-by-rbuild

### Tweet 3

How it works:

1. RUN - Execute your skill
2. EVALUATE - Rate the output (1-5)
3. LEARN - Skill absorbs what worked
4. EVOLVE - New version with improvements

The result? Skills that compound. Session 50 is better than session 1.

### Tweet 4

Works with your existing skills.

Already have prompts and skills? Import them:

```
./import-skill.sh ~/my-skills/code-review
```

Wraps your skill in the self-improving structure. Original behavior stays intact. Now it can learn.

### Tweet 5

Choose how skills evolve:

A) Post-execution prompts - "Rate 1-5. Update?"
B) Wrap-up integration - Review at end of session
C) Manual - You control when to evolve
D) All features - Maximum learning

Setup wizard lets you pick.

### Tweet 6

The skill I use every session: wrap-up

4 phases:
- Ship It (commits code)
- Remember It (saves learnings)
- Review & Apply (Claude fixes its own mistakes)
- Publish It (drafts content)

Self-improving tooling with zero effort from me.

### Tweet 7

Static skills are a waste.

Every time you manually improve a prompt, that learning should be:
- Captured
- Versioned
- Shared

The people who compound their AI skills will outpace everyone else.
